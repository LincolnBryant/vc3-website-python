Generating SSH and Access:

Do you already have an SSH Public Key?
  1. To find out, open your terminal and type: ls ~/.ssh
  2. If there is a .pub extension, such as id_rsa.pub, that is your SSH Public Key.
  3. Type: cat ~/.ssh/id_rsa.pub in order to see your key
If you do not have an SSH Public Key:
  1. In a terminal, type:
    ssh-keygen -t rsa
  2. Hit enter for the default location, and enter a new password
  3. In order to see your key type:
    cat ~/.ssh/id_rsa.pub
  4. Use your mouse to select everything that is printed to the screen, the format should look like:
    ssh-rsa AAAAB3N....M7Q== yourusername@yourmachine
  5. Copy the selection to the clipboard
  6. Paste the contents of the clipboard in the corresponding box on your VC3 Profile
To access your node:
  1. In a terminal, type:
    ssh -i ~/.ssh/id_rsa yourvc3username@IPofTheHeadNode
  2. Members of your project can login using their SSH-Keys and VC3-Usernames
Once an SSH-Key appears (you will only need to do this once per allocation):
  1. In a terminal, type:
    ssh allocation.accountname @resource.accesshost
  2. Enter your password for resource.accesshost for access
  3. Next you will need to replace PUT-THE-ALLOCATION-SSH-KEY-HERE with the actual allocation key (note that it needs to be quoted), type:
    echo 'PUT-THE-ALLOCATION-SSH-KEY-HERE' >> ~/.ssh/authorized_keys
  4. Finally, type:
    chmod 633 ~/.ssh/authorized_keys
